# 欧冶世界监控 — 网站实用性审查报告

> 审查视角：**欧冶半导体员工日常实用工具**
> 审查人：IT编程专家 + 网站架构师 + 视觉专家
> 日期：2026-06-05
> 方法：agent-browser 实地操作 + 全页面代码分析

---

## 总体评价

当前网站具备一个**良好的视觉框架和基础架构**，但从"员工日常真实使用"的角度来看，存在 **3 个致命问题和 5 个重要缺陷**，导致网站目前**难以成为员工真正依赖的工具**。

核心矛盾：**网站做了许多"看起来很 cool"的东西（炫酷地图、滚动行情、动画效果），但最基础的内容可靠性（准确性、相关性、时效性）没有过关。**

---

## 🔴 P0 — 致命问题（导致网站不可用）

### 1. 内容相关性严重失控（75%+ 无关内容）

**现状**：半导体标签页下 8 条新闻中 **6 条完全不相关**：
| 新闻标题 | 相关性 |
|---------|--------|
| "The Best Pool Accessories to Upgrade Your Summer" | ❌ 毫无关系 |
| "Doo Doo Water and a Few Needles: Inside the Mystery..." | ❌ 毫无关系 |
| "AI Has Come for Serif Fonts" | ❌ 无关 |
| "30% Off Canon Promo Codes" | ❌ 促销广告 |
| "Why Apple Might Put Cameras Into Its Next AirPods" | ❌ 消费电子 |
| "OECD data show China-based semiconductor makers..." | ✅ 核心相关 |
| "Metalenses move from lab concept to robotics..." | ⚠️ 部分相关 |

**根因**：`EXTENDED_NEWS_SOURCES` 包含了通用新闻网站的**全站 RSS**（而非分类频道）：

```
第 155 行：The Verge → https://www.theverge.com/rss/index.xml  ← 全站！包含所有科技/非科技内容
第 156 行：Wired   → https://www.wired.com/feed/rss              ← 全站！包含生活/购物/文化
第 150 行：36氪    → https://36kr.com/feed                       ← 全站！包含所有频道
```

这些通用 RSS 源的行业属性被设为 `industry: 'all'`，导致所有内容（包括泳池配件、Manhole 奇闻）渗透到**每一个标签页**（半导体/智能汽车/机器人/AI）。

**影响**：
- 员工打开半导体标签 → 看到泳池配件 → **立即失去对网站专业性的信任**
- "这网站连新闻都选不对，我怎么信它的数据？"

**修复方向**（5条并行）：
1. **改用分类 RSS**：The Verge → `theverge.com/tech/rss`，Wired → `wired.com/category/business/feed`
2. **增加标题关键词白名单**：标题必须包含 [芯片/半导体/NPU/SoC/AI 芯片/自动驾驶/机器人/Nvidia/Qualcomm/TSMC/Intel/AMD...] 等至少一个关键词
3. **增加标题黑名单过滤**：不含 [pool/accessories/promo code/font/serif/recipe...] 等消费类关键词
4. **对通用源降权/限量**：The Verge/Wired/36氪 等通用源每轮最多取 2 条，且必须经过白名单过滤
5. **`industry: 'all'` 风险标记**：带此标签的内容在所有 tab 显示前应做二次关键词匹配

### 2. 行业指标数据 "LIVE" 假象 — 实际 3 周旧

**现状**：页面顶部显示 "● LIVE" 状态灯和当前时间，但指标数据是死的：

| 标签页 | 指标 | 显示值 | 数据日期 | 陈旧程度 |
|--------|------|--------|---------|---------|
| 半导体 | 费城半导体指数 SOX | 4,856.32 | 2026-05-14 | **22天前** |
| 智能汽车 | 中证智能汽车指数 | 2,892.45 | 2026-05-15 | **21天前** |
| 机器人 | 中证机器人指数 | 2,156.89 | 2026-05-15 | **21天前** |
| AI | 中证人工智能指数 | 4,521.89 | 2026-05-15 | **21天前** |

**根因**：`generateDynamicIndices()`（dataService.ts:385-391）仅给 `BASE_INDICES` 打了一个新时间戳，**数值是 hardcode 的静态数据**：

```typescript
function generateDynamicIndices(): IndustryIndex[] {
  const now = new Date().toISOString()
  return BASE_INDICES.map(index => ({
    ...index,           // ← value/change/changePercent 全是静态值！
    timestamp: now      // ← 只更新了时间戳
  }))
}
```

而底部滚动行情却显示了**真实的**实时数据（SOX 4,945.77 而非顶部卡片的 4,856.32）——同一个页面存在数据矛盾！

**影响**：
- "LIVE" 标签 → **误导**（数据 3 周没更新）
- 顶部和底部同一指数显示不同数字 → **信任崩塌**
- 员工做决策前会想："我到底该信哪个数？"

**修复方向**：
1. **接入真实指数 API**：东方财富 `/stock/EM_Index` 接口可获取费城半导体 SOX 实时数据（已有类似的 JSONP 股票行情机制，可将 INDEX 也接入）
2. **或用底部行情数据统一**：顶部卡片至少应与底部滚动行情使用同一数据源
3. **没有实时数据时显式标注**：`状态：每日更新（最后更新 2026-05-15）` → 透明比装酷更重要

### 3. 风险预警和公司新闻模块永久"加载中"

**现状**：首页右侧的"风险预警"和"公司新闻"模块显示 `⏳ 数据加载中... 正在从RSS源获取最新警报数据`，这是一个**无法自愈的加载失败状态**。

而在标签页版（半导体/汽车/机器人/AI）中，这些模块被**完全隐藏了**，只有主页显示——但主页也同样加载不出数据。

**影响**：
- 这是员工最需要的信息（什么风险需要关注？欧冶自己有什么动态？）
- 如果永远加载不出 → 不如删掉，保留只会降低信任

**修复方向**：
1. 风险预警：基于真实新闻自动生成（已有 `generateSentimentFromNews`），应在页面加载时立即填充而非空等
2. 公司新闻：`fetchCompanyNews()` 的三级降级机制可能存在最终兜底也失败的路径 → 增加硬兜底（展示最近 3 天缓存 + 标注"暂未获取到实时数据"）
3. 增加 `lastError` 展示 + 重试按钮（不依赖全局刷新）

## 🟡 P1 — 重要缺陷（显著降低实用价值）

### 4. 信息架构：5 个标签页内容同质化严重

**现状**：切换"半导体 → 智能汽车 → 机器人 → AI"，新闻列表**高度重叠**。因为 `industry: 'all'` 的内容在所有页面显示，且每个标签页的分类筛选按钮都一样（全部/竞争/市场/科技/政策/供应链）。

**问题**：这 5 个标签页本质上只是**同一数据源换了个标题和顶部指标卡片**。

**员工视角**："我打开'智能汽车'想看汽车芯片的量产消息，但看到的还是半导体新闻。"

**修复方向**：
1. 每个标签页应展示该行业专属内容 + 明确标注"跨行业相关内容"以区分
2. 顶部指标卡片应该是**可折叠的**（员工熟悉后不需要每次都看满屏数字）
3. 对于各标签页共用的 RSS 源，应在前端增加行业过滤：标题/摘要关键词匹配（"自动驾驶/ADAS/车载"→汽车，"芯片/制程/晶圆"→半导体）

### 5. 搜索功能形同虚设

**现状**：搜索"英伟达" → 显示 `暂无该分类数据，正在从RSS源获取...`

代码中搜索逻辑是**前端 JavaScript 字符串匹配**（main.ts:1418-1426），仅在已加载的 `newsData` 中过滤：
```typescript
filteredNews = filteredNews.filter(n =>
  n.title.toLowerCase().includes(q) || n.summary?.toLowerCase().includes(q)
)
```

**问题**：
- 搜索范围仅限当前已加载的前 15 条新闻 → 结果是"暂无"
- 不支持跨标签页搜索 → 在 AI 标签页搜"英伟达"找不到半导体标签页的 NVIDIA 新闻
- 没有搜索历史、没有建议、没有高级筛选（时间/来源/影响级）

**员工视角**："我搜公司名字都搜不到，这个搜索是真还是假的？"

**修复方向**：
1. 扩大搜索范围：不是已显示的 15 条，而是全部缓存的 `newsData`
2. 跨标签页搜索 + 结果标注来源标签页
3. 增加时间范围筛选（今天/本周/本月）
4. 搜索历史（localStorage 存储最近 10 次搜索）

### 6. 没有导出/分享/收藏/提醒机制

**现状**：网站纯展示，没有交互能力。

**员工痛点**：
- "有一条重要新闻 → 我想发给同事 → 只能复制链接手动发"
- "我想每天早上自动收到芯片行业简报 → 没有这个功能"
- "上次看到一条政策解读，想再找找不到 → 没有搜索历史和收藏"

**修复方向（按优先级）**：
1. **一键复制链接** — 每条新闻旁边的 👤 分享按钮
2. **收藏功能** — localStorage 存储，收藏夹页面
3. **日报导出** — 点击即生成今日行业简报（Markdown/PDF）
4. **邮件/企微订阅** — 每日 8:00 自动发送行业简报（需要后端）

### 7. 缺少关键业务场景覆盖

| 缺失场景 | 重要性 | 说明 |
|---------|--------|------|
| **竞品追踪** | ⭐⭐⭐⭐⭐ | NVIDIA/Qualcomm/TI/Mobileye 最新动态、产品路线图、客户赢单 — 当前仅散落在新闻中 |
| **客户动态** | ⭐⭐⭐⭐⭐ | 吉利/赛力斯/北汽/一汽/比亚迪等车企的智驾策略、定点变更 — 完全缺失 |
| **人才市场** | ⭐⭐⭐⭐ | 半导体/AI 行业关键人才流动、薪酬趋势、招聘热点 |
| **政策日历** | ⭐⭐⭐⭐ | 芯片法案、出口管制、补贴申报截止 — 当前仅有散落新闻 |
| **投融资** | ⭐⭐⭐ | 半导体/AI 公司融资轮次 — 当前在新闻中偶有提及 |
| **展会/活动** | ⭐⭐ | CES/MWC/ChinaJoy/北京车展等行业活动日历 |
| **供应链地图** | ⭐⭐⭐ | 晶圆厂产能、关键原材料（稀土/光刻胶）价格、物流中断 |
| **专利/论文** | ⭐⭐ | 竞争对手专利动态、关键技术领域论文趋势 |

### 8. 数据可验证性缺失

**现状**：新闻有来源标注（如"Wired"、"Digitimes"），但：
- 无来源可信度评分展示
- 无原始链接校验状态
- 评论/分析类内容与新闻报道混在一起无区分
- "风险预警" 的生成依据不透明

**员工视角**："这个风险预警是系统根据什么生成的？我应该信还是不信？"

**修复方向**：
1. 每条新闻旁展示来源可信度评分（`⭐ 82` 已有系统，但未展示在新闻条目上）
2. 区分"新闻"vs"分析/观点"via 标签
3. "风险预警"模块增加"生成依据"折叠面板

## 🟢 P2 — 一般建议（锦上添花）

### 9. 暗色模式一致性问题

- 搜索框 Placeholder 文字在暗色背景下对比度不足
- 新闻链接 hover 态不明显
- 全局态势地图的标记在缩放时没有 tooltip

### 10. 加载性能

- 首页 world-110m.json (TopoJSON) 约 500KB → 首次加载较慢
- 可做：地图数据预加载、首屏关键 CSS 内联、图片资源懒加载

### 11. 移动端适配完全缺失

- 当前仅在 1920×1080 大屏上可正常使用
- 手机/平板打开 → 布局完全破碎
- 员工用手机快速查行情 → 不可用

### 12. 缺少使用说明/指南

- 新员工首次打开不知道这网站能做什么
- 无引导 / 新手提示 / 使用场景说明

## 💡 核心洞察

这个网站已经有一个**很好的骨骼**（代码架构、视觉设计、数据管道），但"肉"（实际内容）不够结实。

> **如果你只能做一件事**：修复 P0-1（新闻内容筛选）。员工打开网站第一眼看到的是内容——内容不可信，所有炫酷的动画和可视化都是白费。

> **如果能做两件事**：加上 P0-2（行业指数真实数据）。"LIVE" 标签 + 3 周旧数据 = 对专业用户来说是致命的信任危机。

> **如果能做三件事**：加上 P0-3（修复模块加载失败）。不工作的功能不如没有——它会让人怀疑整站的数据都不可靠。
